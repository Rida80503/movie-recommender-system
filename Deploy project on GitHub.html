Got it 👍 Let’s go step by step. Deploying a project on GitHub means uploading (pushing) your project files from your local system to a repository hosted on GitHub.

🔹 Step 1: Install Git

  1. Download and install Git → https://git-scm.com/downloads

  2. After installation, open Git Bash or your terminal/command prompt.

  3. Configure Git with your GitHub account:

    bash
    git config --global user.name "YourGitHubUsername"
git config --global user.email "youremail@example.com"

🔹 Step 2: Create a Repository on GitHub

  1. Go to GitHub.

  2. Click the + icon (top-right) → New repository.

  3. Enter:

    • Repository name (e.g., my-project)

    • Description (optional)

    • Keep it Public or Private

    • ✅ Add a README (optional)

  4. Click Create repository.

🔹 Step 3: Initialize Git in Your Project

Inside your project folder, open terminal and run:

bash
git init

🔹 Step 4: Connect Local Project to GitHub

Copy the repository URL from GitHub (looks like https://github.com/username/my-project.git).

Run:

bash
git remote add origin https://github.com/username/my-project.git

🔹 Step 5: Add and Commit Your Code

bash
git add .
git commit -m "First commit - added project files"

🔹 Step 6: Push to GitHub

bash
git branch -M main
git push -u origin main

Now your project is live on GitHub 🎉

✅ If your project is a website (HTML, CSS, JS, Streamlit, React, etc.), you can also deploy it via GitHub Pages or Streamlit Cloud.

So the filename is not exactly requirements.txt, it’s probably something like:

php-template
requirements.txt<space>

🔹 2. What does the M mean in VS Code?

In VS Code, the M means Modified — the file has unsaved changes compared to the last saved state.

So:

  • requirements.txt is modified (you typed packages but haven’t saved it yet).

  • That’s why pip can’t find it → because VS Code hasn’t written the file to disk yet.

🔹 3. How to fix this

  1. In VS Code, press Ctrl+S (save the file).

    • The M will disappear.

  2. Now run:

    bash
    pip install -r requirements.txt

This time it will work because the file is saved and pip can read it. ✅

👉 Extra tip: To double-check the file really exists, run in terminal:

bash
dir

(Windows) or

bash
ls

(Mac/Linux). You should see requirements.txt in the list.
